Architecture : comprendre les 3 flux

L'espace client a 3 types de contenu qui arrivent sur O2switch par 3 chemins differents :

Code PHP, config, CSS
→
git push + SSH
O2switch
deploy.sh
Resumes, donuts, docs, mission.json
→
FTP
O2switch
hebdo.sh
Google Docs partages
→
Drive sync → hebdo.sh lit doc_id
Liens dans BDD

Regle d'or : le code passe par git, le contenu passe par FTP.

Le code (PHP, templates, CSS) vit dans un repo git — c'est propre, versionne, on peut revenir en arriere. Les docs clients (resumes, PDF, images) n'ont rien a faire dans un repo : ils passent en FTP direct.

Les Google Docs partages (fichiers .gdoc, .gsheet) sont synchro auto sur H:/ via Drive for Desktop. hebdo.sh lit le doc_id dans ces fichiers JSON et construit le lien Google — pas besoin d'API.

Prerequis (configure une seule fois)

1. SSH sans mot de passe

Pour que deploy.sh fonctionne sans taper le mot de passe a chaque fois :

1

Copier ta cle SSH sur O2switch

# Si tu n'as pas encore de cle SSH :
ssh-keygen -t ed25519
# (Appuie Entree a tout, pas besoin de passphrase)

# Puis copie-la sur le serveur (mot de passe demande une derniere fois) :
ssh-copy-id qufi1696@raquette.o2switch.net

# Teste que ca marche :
ssh qufi1696@raquette.o2switch.net "echo OK"
# Doit afficher "OK" sans demander de mot de passe

2. FTP (~/.netrc)

Le fichier ~/.netrc est un fichier cache dans ton dossier personnel (/Users/sergefornier/.netrc). C'est un standard Unix : quand curl doit s'authentifier en FTP, il va chercher les identifiants la-dedans automatiquement.

2

Configurer ~/.netrc

nano ~/.netrc

Contenu (remplacer le mot de passe) :

machine ftp.qufi1696.odns.fr
login contenu@mission.lapmedigitale.fr
password TON_MOT_DE_PASSE_FTP

Sauvegarder (Ctrl+O, Enter, Ctrl+X), puis securiser :

chmod 600 ~/.netrc
Attention au hostname : le serveur FTP est ftp.qufi1696.odns.fr (le serveur physique O2switch), pas mission.lapmedigitale.fr (qui est juste le domaine web). Le login reste contenu@mission.lapmedigitale.fr.
3

Tester la connexion FTP

# Lister la racine du site
curl -s -k --ftp-ssl-control --netrc "ftp://ftp.qufi1696.odns.fr/"

Tu devrais voir les dossiers missions/, includes/, index.php, etc.

Si ca liste des fichiers → FTP OK.
Si "Login denied" → verifier le mot de passe dans ~/.netrc. Attention : le mot de passe est copie tel quel, les caracteres speciaux peuvent poser probleme. En cas de doute, changer le mot de passe FTP dans cPanel pour un mot de passe sans caracteres exotiques.

3. Python + bcrypt

4

Verifier / installer

python3 -c "import bcrypt; print('OK')"

# Si "ModuleNotFoundError" :
pip3 install bcrypt
bcrypt sert a generer les mots de passe hashes pour les comptes utilisateurs. C'est utilise par open-ec.sh.

Les 3 scripts

Tous dans _scripts/. Toujours lancer depuis la racine du repo :

cd ~/Documents\ \(H\)/01\ -\ TRAVAIL/LAB\ IA/projets/espace-client

deploy.sh — Deployer du CODE

Quand : apres une modification PHP, config, template, CSS, ou mission.json.

Usage

./_scripts/deploy.sh "Description courte de la modif"

Ce qu'il fait : git add → commit → push → SSH git pull → copie src/* a la racine.

Ne touche PAS au contenu (resumes, docs). Pour ca → hebdo.sh.

Message de commit : reste simple et utile — ce qui a change en une ligne. Exemples : "Ajout mission Bonnavion", "Fix affichage phase accueil", "Correction lien doc partage". Pas besoin de numero de version ni de date, git s'en charge.
Alternative VS Code : tu peux faire le commit/push via le panneau Source Control de VS Code (tu sais faire), puis lancer uniquement la partie serveur :
ssh qufi1696@raquette.o2switch.net "cd ~/mission.lapmedigitale.fr && git pull origin main && cp -r src/* . && cp src/.htaccess ."

open-ec.sh — Ouvrir une NOUVELLE mission

Quand : nouvelle mission a rendre visible dans l'espace client.

Usage

./_scripts/open-ec.sh SLUG email@client.fr "Nom Complet"

Exemples :

./_scripts/open-ec.sh bonnavion-2025 andre@bonnavion.fr "Andre Bonnavion"
./_scripts/open-ec.sh mms-2026 sebastien.bay@mms-ra.fr "Sebastien Bay"

Ce qu'il fait :

  • Cree missions/SLUG/ + resumes/ sur O2switch (FTP)
  • Uploade mission.json (depuis H:/Missions/missions/SLUG-ms/resumes-ec/)
  • Genere le SQL d'insertion utilisateur (hash bcrypt + mot de passe temporaire)
Apres : executer le SQL sur phpMyAdmin (ou via /admin.php) → tester le login → envoyer les credentials au client.

hebdo.sh — Mise a jour HEBDOMADAIRE

Quand : chaque semaine, apres avoir redige les resumes.

Usage

./_scripts/hebdo.sh S20

Ce qu'il fait :

  • Scanne toutes les missions actives (H:/Missions/missions/*-ms/resumes-ec/)
  • Upload FTP : resumes (2026-S20.md), donuts (donut-S20.png)
  • Re-uploade mission.json si modifie dans les 7 derniers jours
  • Scanne partage-shared/ :
    • .gdoc/.gsheet/.gslides → extrait doc_id → SQL avec lien Google Drive
    • Fichiers classiques (PDF, images, xlsx, docx) → upload FTP sur O2switch
  • Genere un fichier SQL dans sql/hebdo-SXX-docs.sql (anti-doublons)
Apres : si le script affiche "SQL genere", ouvrir le fichier indique (sql/hebdo-SXX-docs.sql) → copier le contenu → coller dans phpMyAdmin → Executer. Si rien a inserer, c'est fini.
Pourquoi le SQL n'est pas execute automatiquement ? On n'a pas de connexion directe a la base MySQL depuis ton Mac. Le script genere le SQL, tu le valides et l'executes sur phpMyAdmin. Ca te laisse le controle sur ce qui entre en base. (Automatisable plus tard via SSH si besoin.)

Routine hebdo (checklist)

  • Rediger le resume H:/Missions/missions/SLUG-ms/resumes-ec/2026-SXX.md
  • (Optionnel) Donut donut-SXX.png
  • Mettre a jour mission.json si jours/phase a change
  • Deposer les nouveaux docs dans partage-shared/direction/ (ou partage-shared/equipe/)
  • Terminal : ./_scripts/hebdo.sh SXX
  • Si SQL genere → executer sur phpMyAdmin
  • Verifier sur mission.lapmedigitale.fr

Troubleshooting

Erreur "index.lock" au git commit

Un fichier verrou traine (git a plante ou un editeur est ouvert). Supprimer :

rm -f .git/index.lock

Puis relancer deploy.sh.

Erreur "Login denied" en FTP

Verifier ~/.netrc : le machine doit etre ftp.qufi1696.odns.fr (pas mission.lapmedigitale.fr). Si le mot de passe contient des caracteres speciaux, le changer dans cPanel pour un mot de passe plus simple.

Certificat SSL inconnu (FileZilla / Cyberduck)

Normal — O2switch utilise un certificat pour raquette.o2switch.net, pas pour ftp.qufi1696.odns.fr. Accepter le certificat et cocher "Toujours faire confiance". La connexion est bien chiffree.

Erreur 451 "Transfer aborted" en upload FTP

Probleme de Pure-FTPd avec TLS sur le canal de donnees. Les scripts utilisent --ftp-ssl-control (SSL sur le login uniquement) pour contourner. Si tu testes manuellement avec curl, utilise --ftp-ssl-control et pas --ftp-ssl.

SSH demande le mot de passe a chaque fois

La cle SSH n'est pas copiee sur le serveur. Lancer :

ssh-copy-id qufi1696@raquette.o2switch.net

Mot de passe demande une derniere fois, puis c'est regle.

"Aucune mission active trouvee" dans hebdo.sh

Le script cherche les dossiers *-ms qui contiennent un sous-dossier resumes-ec/. Si une mission n'a pas ce dossier, elle est ignoree. Creer le dossier : mkdir -p H:/Missions/missions/SLUG-ms/resumes-ec/
