Pourquoi séparer ? Le code (PHP) vit dans un repo git — c'est propre, versionné, on peut revenir en arrière. Les docs clients (résumés, PDF) n'ont rien à faire dans un repo : ils passent en FTP.

Étape 1 — Créer le compte FTP sur O2switch

Tu dois créer un compte FTP dédié qui pointe vers le dossier de l'espace client. Ce compte sera utilisé par open-ec.sh et hebdo.sh.

1

Se connecter au cPanel O2switch

Aller sur ton admin O2switch → cPanel. Chercher la section "Comptes FTP" (ou "FTP Accounts").

2

Créer le compte FTP

Remplir le formulaire :

ChampValeur
Connexion / Logincontenu@mission.lapmedigitale.fr
Mot de passeGénérer un mot de passe fort (le noter !)
Répertoire/home/qufi1696/mission.lapmedigitale.fr
QuotaIllimité
⚠ Le répertoire doit pointer vers la racine du site mission.lapmedigitale.fr, pas vers /home/qufi1696/. Vérifie que c'est bien le cas.
3

Configurer ~/.netrc sur ton Mac

Ouvre le Terminal et tape :

# Créer ou éditer le fichier
nano ~/.netrc

Coller ces 3 lignes (remplacer le mot de passe) :

machine ftp.qufi1696.odns.fr
login contenu@mission.lapmedigitale.fr
password TON_MOT_DE_PASSE_FTP

Sauvegarder (Ctrl+O, Enter, Ctrl+X), puis sécuriser :

chmod 600 ~/.netrc
4

Tester la connexion FTP

# Test rapide : lister le contenu du dossier missions/
curl -s -k --ftp-ssl --netrc "ftp://ftp.qufi1696.odns.fr/missions/"

Tu devrais voir les dossiers fl-metal-2026 et rs-2026 (ceux déjà en prod).

✓ Si ça liste des fichiers, le FTP est prêt.
✗ Si erreur "Login denied" → vérifier le mot de passe dans ~/.netrc.
✗ Si erreur SSL → essayer sans --ftp-ssl ou vérifier dans cPanel que FTPS est activé.

Étape 2 — Installer bcrypt (si pas encore fait)

1

Vérifier / installer

# Tester si bcrypt est installé
python3 -c "import bcrypt; print('OK')"

# Si "ModuleNotFoundError" :
pip3 install bcrypt
bcrypt sert à générer les mots de passe hashés pour les comptes utilisateurs. C'est utilisé par open-ec.sh.

Étape 3 — Premier déploiement (le faire une fois)

On va pousser le code qu'on a produit aujourd'hui, puis ouvrir les deux nouvelles missions.

3a. Pousser le code

1

Ouvrir le Terminal, aller dans le repo

cd ~/Documents\ \(H\)/01\ -\ TRAVAIL/LAB\ IA/projets/espace-client
2

Lancer deploy.sh

./_scripts/deploy.sh "V1.5 — multi-missions Bonnavion + MMS + statut"

Le script va : git add → commit → push → SSH sur O2switch → git pull → copier les fichiers.

✓ Si tu vois "✓ Code déployé", c'est bon.
3

Vérifier

Aller sur mission.lapmedigitale.fr et te connecter en admin. Tu dois voir les 4 missions dans la liste.

⚠ Les missions Bonnavion et MMS apparaîtront dans la liste mais ne seront pas encore cliquables — il faut d'abord les ouvrir en FTP (étape suivante).

3b. Ouvrir les missions en FTP

4

Ouvrir Bonnavion

./_scripts/open-ec.sh bonnavion-2025 andre@bonnavion.fr "André Bonnavion"

Le script va :

  • Créer missions/bonnavion-2025/ sur O2switch
  • Uploader mission.json
  • Générer le SQL + mot de passe temporaire
📝 Note le mot de passe temporaire affiché dans le terminal !
5

Ouvrir MMS

./_scripts/open-ec.sh mms-2026 sebastien.bay@mms-ra.fr "Sébastien Bay"
⚠ Vérifie l'email de Sébastien Bay avant de lancer — je n'ai pas l'email confirmé.
6

Exécuter les SQL

Deux options :

Option A (recommandé) : aller sur /admin.php, déplier "Ajouter un utilisateur", et créer les comptes directement dans l'interface.

Option B : copier le SQL affiché par le script → coller dans phpMyAdmin → exécuter.

7

Tester

  • Se connecter en tant qu'André Bonnavion → voit la mission Bonnavion
  • Se connecter en tant que Sébastien Bay → voit la mission MMS
  • Se connecter en admin → voit les 4 missions, actives triées en premier

Étape 4 — Usage hebdomadaire (la routine)

Chaque semaine, après avoir rédigé tes résumés :

1

Rédiger le résumé

Créer le fichier dans le dossier mission sur H:/ :

# Exemple pour FL Metal, semaine 20
H:/Missions/missions/FL_METAL_2026-ms/resumes-ec/2026-S20.md
2

Lancer hebdo.sh

cd ~/Documents\ \(H\)/01\ -\ TRAVAIL/LAB\ IA/projets/espace-client
./_scripts/hebdo.sh S20

Le script scanne toutes les missions, uploade les nouveaux fichiers, et génère le SQL si des documents Google Docs ont été ajoutés dans partage-shared/.

3

SQL si nécessaire

Si le script affiche "SQL généré", exécuter le fichier indiqué sur phpMyAdmin.

Si rien à insérer, c'est fini.
